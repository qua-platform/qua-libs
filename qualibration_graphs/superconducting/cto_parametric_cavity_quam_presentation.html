Key decisions (why, not only what)

  • Single-file components. Kept StorageCavity, ParametricDriveMW, ParametricCavityTransmon, and Quam in my_quam.py so calibration imports stay one hop (from quam_config import Quam) and review surface stays small.
  • BaseQuam root. Chosen over FluxTunableQuam (no flux-tunable transmon / no z in v1) and over tightening to FixedFrequencyQuam while still adding cavity and parametric_drive fields on the qubit type — explicit Dict[str, ParametricCavityTransmon] on the root avoids stock pair / flux init assumptions until needed.
  • Manual generate_quam_parametric_cavity. Skipped Connectivity / allocate_wiring because no stock pattern maps “cavity XY + parametric MW on flux line” without misleading semantics.
  • Port deduplication via #/ports/... strings. Readout and transmon XY share MW-FEM O1; QuAM forbids assigning the same MWFEMAnalogOutputPort instance to two parents — references resolve to one port object in ports.
  • Initial RF_frequency on channels at generate time. Required so ReadoutResonatorMW can resolve inferred_intermediate_frequency; avoids hung or warning-heavy generate_config() during first load.
  • Cluster vs controller id split. network.cluster_name = "CS_4" for QMM discovery; controller_id = "con1" (and FEMPortsContainer keys) for QUA controllers — QOP validates resource names separately from dashboard labels.
  • active_qubit_names. Explicit list in generate plus populate fallback — empty list breaks Qualibration nodes (zero qubits → empty stream processing).
  • QUAM_STATE_PATH → quam_config/quam_state. Co-locates machine JSON with the package that owns the schema; repeatable CI / laptop runs.
  • MW-FEM only in v1. No LF-FEM / DC flux; all difference tones (0.2 / 1.5 / 1.7 GHz) on O4 Band 1 per hardware discussion.

Physics & connectivity

Three coupled modes; two physical lines to the chip (microwave feed and “flux” line carrying parametric RF).

QUAM / code

What we built

my_quam.py (single module)

  • StorageCavity — QuamComponent with xy: XYDriveMW, lifetimes, f_01.
  • ParametricDriveMW — extends MWChannel; tracks mode frequencies; detuning helpers (0.2 / 1.5 / 1.7 GHz).
  • ParametricCavityTransmon — extends FixedFrequencyTransmon; adds cavity and parametric_drive (no flux z line in v1).
  • Quam — extends BaseQuam with qubits: Dict[str, ParametricCavityTransmon] (avoids flux-tunable root assumptions while keeping fixed-frequency qubit semantics).

Scripts

  • generate_quam_parametric_cavity.py — Manual FEM wiring (no generic wirer template); shared O1 via #/ports/... references; saves QUAM state.
  • populate_quam_parametric_cavity.py — Frequencies, bands, powers, readout / XY / cavity / parametric pulses; writes qua_config.json when run.
